<analysis>**original_problem_statement:**
Build a mobile-responsive community fund management web app, Hussain Maat Foundation, with role-based access for Members, Admins, and a Super Admin. There is no separate admin panel.

PRODUCT REQUIREMENTS:
- **User Management:**
    - Users sign up with a Pakistani phone number and a 4-digit PIN. First name and last name are collected.
    - Super Admin phone is +923142256184.
    - Admins can be promoted by the Super Admin.
    - Super Admin should be able to add new members (first name, last name, optional phone number).
    - Admins should have options to update, delete, or disable users.
- **Fee Structure & Collection:**
    - Admin defines a single active monthly fee for the entire year (e.g., PKR 200/month).
    - The system should calculate the corresponding yearly fee (e.g., PKR 2400/year).
    - Members can pay more than the defined fee as a donation.
    - Admins collect fees on behalf of members. The screen should allow selecting a member, the year, and choosing to pay for specific months or the entire year.
    - The label for fee submission should be Collect Fee.
- **Fund Allocation:**
    - Admins can allocate funds for cases like school fees, health expenses, etc.
    - When allocating funds, the recipient's name and phone number must be recorded.
    - A history screen should list all fund allocations with columns: Case ID, Amount, Recipient Name, Allocated Date, Allocated By.
- **Approval Workflow:**
    - Fee submissions and fund allocations require approval from at least 2 other admins.
    - Approval statuses: Pending, Approved, Rejected.
    - A history of approvals must be maintained.
- **Dashboards & UI:**
    - **Member Dashboard:** Show total pending/paid fees (monthly/yearly), total collections, expenses, and remaining amount.
    - **UI/UX:**
        - The app must be bilingual (English and Urdu) with a language switcher.
        - Use the provided Hussain Maat Foundation logo and name for branding on all screens. The logo on inner pages should be a clickable home link.
        - Redesign the login screen to be more intuitive.
        - Fee Collection and Fund Allocation screens should use a tabular format with one-click approve/reject actions.
        - Implement bulk actions where applicable.
        - Add a search option for members on the fee collection and fund allocation screens.
        - Display the actual admin's name and number instead of a generic Admin label.
        - Add a Back button on all inner screens.
- **Notifications:**
    - In-app notifications for fee submission (submitted/approved/rejected) and fund allocation (approved/rejected).

**User's preferred language**: English
